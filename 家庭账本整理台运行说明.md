# 家庭账本整理台运行说明

## 1. 这个程序是什么

家庭账本整理台是一个本地运行的 HTML 产品，用来把多个账单文件整理成两个结果：

1. 统一格式的交易明细
2. 按分类汇总的统计表

目前支持把 `账单文件夹` 里的 Excel / CSV 账单读进来，也支持用户手动选择账单文件。页面会先读取账单，再统一字段、自动分类、让 AI 复核拿不准的项目，最后生成结果和导出文件。

## 3. 推荐运行方式

推荐用本地服务打开，这样可以自动读取 `账单文件夹`，也可以调用 AI。

在 PowerShell 里进入目录：

```powershell
cd d:\财务整理
```

如果要启用 AI，需要先设置环境变量：

```powershell
$env:AI_API_KEY="你的AI密钥"
$env:AI_BASE_URL="https://api.aispeeded.com/v1"
$env:AI_MODEL="claude-opus-4-8"
$env:AI_API_FORMAT="chat_completions"
```

`server.py` 兼容 `https://api.aispeeded.com` 和 `https://api.aispeeded.com/v1` 两种写法；当前默认按 `/v1/chat/completions` 调用。

如果要切换到 APIyi 的 Anthropic Messages 渠道，用这一组：

```powershell
$env:AI_API_KEY="你的AI密钥"
$env:AI_BASE_URL="https://api.apiyi.com/v1"
$env:AI_MODEL="claude-opus-4-6"
$env:AI_API_FORMAT="anthropic_messages"
$env:ANTHROPIC_VERSION="2023-06-01"
```

APIyi 渠道会调用：

```text
https://api.apiyi.com/v1/messages
```

启动本地服务：

```powershell
python server.py 8765
```

然后在浏览器打开：

```text
http://127.0.0.1:8765/家庭账本整理台_单文件.html
```

## 4. 也可以直接打开 HTML 吗

可以，但能力会少一些。

直接双击 `家庭账本整理台_单文件.html` 时：

- 可以手动选择账单文件
- 可以本地解析账单
- 可以本地分类
- 可以导出 CSV / Excel

但直接打开 HTML 时：

- 不能自动读取 `账单文件夹`
- 不能调用 `/api/list-bills`
- 不能调用 AI 训练助手
- 不能调用 AI 复核接口

所以正式使用建议走本地服务方式。

## 5. 页面使用流程

### 5.1 导入账单

打开页面后，首页会显示当前 `账单文件夹` 里发现的账单。

用户可以：

- 点击 `导入并整理`，直接读取账单文件夹里的文件
- 点击 `选择账单`，手动选择一个或多个账单
- 点击 `选择文件夹`，手动选择整个账单目录

### 5.2 中间处理过程

导入后不会立刻跳结果页，而是进入处理页：

1. `读取账单文件`
   读取 Excel / CSV，识别不同工作表。

2. `统一字段格式`
   把不同账单里的日期、金额、方向、商户、摘要、来源统一成固定字段。

3. `规则分类`
   先按用户记忆和固定规则分类，再套默认规则。

4. `AI 复核`
   只把规则拿不准的交易交给 AI 判断，不会把所有交易都重新交给 AI。

5. `生成结果`
   生成统一交易明细、分类统计、待确认清单。

## 6. AI 复核到底在做什么

AI 复核不是训练，也不是重新解析 Excel。

它只处理系统本地规则拿不准的交易。比如系统先识别出 233 条账单，其中 60 条不确定，就会显示：

```text
AI 正在复核第 1/3 批，共 60 条不确定交易
```

意思是：

- 最多挑 60 条不确定交易给 AI 看
- 每批大约 20 条，所以分成 3 批
- AI 根据商户、摘要、金额、日期、渠道给分类建议
- 置信度高的自动写入分类
- 置信度低的仍然保留在 `待确认`

这样做是为了避免 AI 一次处理太多导致不稳定，也避免 AI 把不确定内容强行分类。

## 7. 分类规则顺序

系统目前大致按这个顺序分类：

1. 当天消费记忆
   例如用户说“今天刚刚消费 200 元，计入社交支出”，月底看到同日期同金额会优先匹配。

2. 固定对手方规则
   例如用户指定“上海宋伊份以后固定为日常消费”。

3. 默认规则
   例如退款/收入、餐饮大于 100、孩子相关、水电费、交通娱乐等。

4. AI 复核
   只复核前面规则拿不准的项目。

5. 用户手动确认
   仍不确定的项目进入待确认，由用户选择分类，也可以选择记住规则。

## 8. 输出内容

页面上有四个主要视图：

- `交易明细`
  查看统一后的所有账单明细，可按分类筛选，也可以搜索商户、摘要、备注。

- `分类统计`
  查看每个分类的消费金额、笔数和占比。

- `待确认`
  查看系统和 AI 都没有完全确定的交易，由用户手动确认。

- `训练规则`
  用户可以告诉系统自己的分类习惯，比如固定对手方规则或当天消费记忆。

## 9. 导出文件

页面支持三个导出动作：

- `明细 CSV`
  只导出统一交易明细。

- `统计 CSV`
  只导出分类统计。

- `导出完整 Excel`
  导出一个完整 Excel，包含统一交易明细和分类统计。

统一交易明细字段固定，主要包括：

- 交易日期
- 记账日期
- 收支方向
- 金额
- 标准分类
- 是否待确认
- 分类置信度
- 商户/对方
- 交易摘要
- 用途/备注
- 支付渠道
- 账户/卡号
- 分类依据
- 来源文件
- 来源行号

## 10. AI 接口说明

`server.py` 提供两个 AI 接口：

```text
POST /api/parse-training
```

用于训练助手理解用户输入，比如：

```text
今天刚刚消费了 200 元，计入社交支出
```

或：

```text
以后上海宋伊份固定为日常消费
```

另一个接口是：

```text
POST /api/review-transactions
```

用于 AI 复核不确定交易，返回分类建议、置信度和原因。

注意：不要把 AI 密钥写进 HTML 文件，也不要写进 Markdown 文档。运行时通过环境变量传给 `server.py`。

## 11. 常见问题

### 页面打开后没有自动发现账单

确认是否是通过本地服务打开：

```text
http://127.0.0.1:8765/家庭账本整理台_单文件.html
```

如果是直接双击 HTML，页面无法自动读取本地目录，只能手动选择账单。

### AI 没有复核成功

检查：

- `AI_API_KEY` 是否设置
- `AI_BASE_URL` 是否正确
- `AI_MODEL` 是否正确
- 网络是否能访问 AI 接口

即使 AI 失败，账单解析、规则分类和导出仍然可以继续运行，只是不确定项目会留在 `待确认`。

### 为什么还有很多待确认

这是正常的。系统不会把低置信度交易强行分类。

用户可以在 `待确认` 里选择分类，并选择记住。后续同类商户或同日同金额消费会优先按用户规则处理。

### 导出的 Excel 在哪里

浏览器会按默认下载规则保存。通常在系统的 `下载` 文件夹里。

## 12. 当前运行逻辑总结

一句话版本：

```text
用户导入账单 → 本地解析和统一字段 → 本地规则先分类 → AI 只复核不确定项 → 用户确认剩余项目 → 导出统一明细和分类统计
```

这个设计的核心是：账单读取和导出尽量稳定地在本地完成，AI 只参与“理解用户说的话”和“判断拿不准的分类”。
